# 信用债研究与决策辅助框架蓝图

目标不是做一个“神谕”，而是做一个能持续研究、验证、解释和辅助决策的信用债系统。

## 一、机构通常怎么做

公开资料能看到几个共同点：

- 头部量化机构会把机器学习放进完整投资流程，而不是只押注单一模型。AQR 公开说明其机器学习工作强调“可实施”并与长期积累的信号体系互补。
- 信用投资机构会用机器学习增强信用估值、风险控制和多因子组合。Robeco 公开介绍过把机器学习用于信用债价值投资和风险控制。
- 大型资管和银行会更重视组合风险、压力测试、情景分析和统一数据平台。BlackRock 的 Aladdin 就是偏组合管理、风险和数据洞察的技术平台。

所以我们也不应该只做“预测明天涨跌”。更好的目标是：预测、解释、风控、情景、组合建议五件事一起做。

## 二、我们自己的分层架构

第一层：数据湖

- DM API 是后续唯一日常数据入口，覆盖宏观、利率、信用债、资金面等数据。
- CSV 导入只作为历史迁移、临时补数和调试工具，不再作为主线。
- 统一日期、频率、缺失值、异常值。
- 保留原始数据和处理后数据，避免以后无法复现。
- 对低频宏观指标必须使用 `publish_time` 或保守滞后规则，避免数据穿越。

第二层：标签和任务

- 短周期方向：未来 1、5、10、20 个交易日。
- 收益率变化幅度：回归任务。
- 信用利差变化：评级和期限分层。
- 风险状态：扩张、收缩、拥挤、流动性压力。

第三层：神经网络实验台

- 长短记忆网络：稳健基线。
- 门控循环网络：轻量基线。
- 时序卷积网络：更适合显卡并行，建议重点测。
- 注意力模型：适合更长窗口和更多指标，但要防过拟合。
- 后续可加多任务模型，一次输出多个预测周期。

第四层：验证系统

- 严格时间切分。
- 滚动训练和滚动测试。
- 混淆矩阵、方向胜率、主动信号比例。
- 简化久期收益 proxy。
- 加入交易成本、滑点、久期和流动性约束后再看真实可用性。

第五层：决策辅助

- 每日输出：看多、看空、震荡概率。
- 信号强度：最高概率和第二高概率的差。
- 风险提示：模型分歧、样本外失效、近期误判增加。
- 人工复核：政策、信用事件、供给冲击、资金面异常。

## 三、下一阶段优先级

1. 做 DM API 每日更新与最新预测流水线。
2. 扩充 DM 指标配置，把信用债收益率、信用利差、资金面和更多宏观指标纳入同一数据源。
3. 加滚动回测，不只做一次训练测试切分。
4. 加多周期预测，同时看 1、5、20 个交易日。
5. 加解释模块，先做置换重要性，再考虑 SHAP。
6. 做一个本地仪表盘，把概率、回测、特征重要性放到同一屏。
