# DM 曲线历史长度核验

生成日期：2026-06-24

## 结论

本次不是看本地缓存，而是直接向 DM API 请求 `2011-01-01` 至 `2026-06-24` 的历史数据。

结果很明确：

- 当前 DM 债券收益率曲线层级里的日频曲线，不管是中短期票据、企业债、政策性金融债(国开)、国债，实际返回基本都是 `2024-09-18` 至 `2026-06-22`，共 434 条，约 1.76 年。
- 这说明短历史不是我们的清洗流程截断，也不是只选了中短票导致；更像是 DM 当前这批日频曲线口径本身在 API 里只给了这一段。
- 非 CFETS 的“中债国债收益率(月频)”能拉到 `2014-01-30` 至 `2026-04-29`，共 148 条，约 12.25 年。
- 中债月频表有 3年、5年、10年、30年，没有 20年；可以用 30年作为长端利率锚，但不能直接替代 20年信用债目标。
- `有担保企业债(AA+)到期收益率:3.0年` 这个候选指标本次 API 无返回数据，暂时不适合入模。

## 本次核验的口径

日频候选：

- 中短期票据 AAA、AAA+、AAA-
- 企业债 AAA-、A
- 有担保企业债 AA+
- 政策性金融债(国开) AAA
- 国债

非 CFETS 候选：

- 中债国债收益率(月频)：3年、5年、10年、30年

## 对模型的影响

短期内，3年、5年、10年、20年信用债目标仍只能用当前 DM 日频曲线训练短样本模型；这部分历史不足会限制神经网络的稳定性。

下一步更合理的做法是：

1. 把中债月频 3年、5年、10年、30年作为长历史利率环境特征接入。
2. 继续在 DM 字典里寻找非 CFETS 的信用债、城投债、产业债、信用利差、估值收益率等口径。
3. 如果 DM 找不到更长的日频信用债目标，就把模型拆成两层：长历史月频/周频方向模型 + 短历史日频执行模型。
