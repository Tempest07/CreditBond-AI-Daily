# 信用债走势预测LSTM模型框架说明文档


## 一、项目概述
### 1.1 项目目标
基于深度学习LSTM（长短期记忆网络），构建端到端的信用债走势预测框架，通过输入宏观经济指标、债券市场数据等原始数据，自动预测未来N天信用债收益率的“看多/看空/震荡”趋势，为信用债投资决策提供数据支持。

### 1.2 核心价值
- 减少人工干预：无需人工构造特征（如信用利差、趋势指标），模型自动从原始数据中学习有效模式；
- 时序适应性强：LSTM网络擅长捕捉时间序列中的长期依赖关系，适合债券市场“历史数据影响未来走势”的特性；
- 落地性高：提供从数据获取到预测输出的全流程自动化工具，可直接对接实际投资场景。


## 二、核心功能与适用场景
### 2.1 核心功能
1. 数据自动化处理：支持从原始数据（或Wind接口）自动清洗、对齐、补全，生成标准化数据集；
2. 端到端模型训练：基于清洗后的原始数据，自动生成预测标签、构建时序窗口，训练LSTM模型；
3. 多场景预测：输入最新数据后，输出未来N天的走势判断（看多/看空/震荡）及对应概率；
4. 效果评估：提供金融场景化评估指标（如看多/看空F1分数、模拟实盘收益），而非单纯依赖准确率。

### 2.2 适用场景
- 信用债投资决策辅助（如持仓调整、交易时机判断）；
- 债券市场趋势监控（实时跟踪模型对市场的判断）；
- 宏观经济与债券市场关联分析（模型隐含的指标影响权重）。


## 三、技术框架与流程说明
### 3.1 整体架构
（架构逻辑：数据层→处理层→模型层→应用层）
- 数据层：原始数据（宏观指标、债券收益率等）存储与获取；
- 处理层：数据清洗、标签生成、时序窗口构建；
- 模型层：LSTM网络训练、评估与优化；
- 应用层：预测接口、结果可视化、投资建议生成。

### 3.2 核心流程（端到端步骤）
#### 1. 数据准备
- 输入：多源原始数据（如GDP、CPI、国债收益率、信用债收益率等）；
- 处理：统一日期格式、对齐频率（如将季度GDP转换为日频数据）、填补缺失值（前向填充）；
- 输出：清洗后的标准化数据集（processed_data/credit_bond_final_data.csv）。

#### 2. 特征工程
- 核心逻辑：无人工特征构造，直接使用原始指标作为模型输入；
- 关键步骤：
  - 生成标签：根据未来N天信用债收益率变化，定义“看多（收益率下跌）/看空（收益率上涨）/震荡（波动微小）”；
  - 时间拆分：按时间顺序拆分训练/验证/测试集（避免数据泄露）；
  - 标准化：对原始特征做Z-score标准化（仅用训练集参数，避免泄露）；
  - 时序窗口：将数据转换为LSTM输入格式（样本数×窗口大小×特征数）；
- 输出：LSTM可直接输入的窗口数据（feature_data/下的.npy文件）。

#### 3. 模型训练
- 网络结构：1层LSTM（32神经元）+ Dropout（20%）+ 输出层（3类概率）；
- 训练策略：早停机制（验证集损失3轮不下降则停止），避免过拟合；
- 输出：训练好的模型（saved_model/best_lstm_model.keras）及评估指标。

#### 4. 预测应用
- 输入：最新原始数据（或通过Wind接口自动获取）；
- 处理：复用训练时的标准化器和窗口逻辑，确保预处理一致性；
- 输出：未来N天走势预测（含概率）及投资建议。


## 四、技术细节说明
### 4.1 数据处理关键参数
| 参数名称         | 含义说明                          | 默认值  | 调整建议                          |
|------------------|-----------------------------------|---------|-----------------------------------|
| predict_days     | 预测未来天数                      | 5天     | 短期预测（1-7天）更可靠           |
| window_size      | LSTM输入窗口大小（过去N天数据）   | 30天    | 数据量多→调大（如60天），反之调小 |
| theta_quantile   | 标签阈值θ的分位数（判断涨跌幅度） | 0.8     | 市场波动大→调低（如0.7）          |

### 4.2 LSTM模型结构
输入层（30天×N特征）→ LSTM层（32神经元，tanh激活）→ Dropout（20%）→ 输出层（3神经元，Softmax激活）
- LSTM层：捕捉原始数据中的时序模式（如“国债收益率上升后信用债收益率滞后上升”）；
- Dropout层：随机失活20%神经元，防止模型过度依赖某类指标（如仅关注GDP）；
- 输出层：输出3类概率（和为1），分别对应“看空/看多/震荡”。

### 4.3 评估指标（金融场景适配）
- 核心指标：看多/看空类的F1分数（平衡精确率与召回率，避免“震荡类占比高导致准确率虚高”）；
- 辅助指标：模拟实盘收益（基于模型预测进行虚拟交易，计算累计收益，直观反映投资价值）。


## 五、使用指南
### 5.1 环境搭建
#### 1. 安装依赖库
在终端执行以下命令：
pip install -r requirements.txt

依赖清单：
- pandas==2.1.4
- numpy==1.26.3
- matplotlib==3.8.2
- scikit-learn==1.3.2
- tensorflow==2.15.0
- joblib==1.3.2

#### 2. 准备原始数据
- 手动准备：将数据放入“1.原始数据文件夹/”，格式为CSV（需包含“date”列及指标列）；
- 自动获取：配置Wind接口（参考wind_data_fetcher.py），定时获取最新数据。

### 5.2 快速启动流程
1. 数据准备：运行“2.数据准备模块/data_preparation.py”，生成清洗后数据；
2. 特征工程：运行“3.特征工程模块/feature_engineering.py”，生成LSTM输入数据；
3. 模型训练：运行“4.LSTM训练模块/lstm_train.py”，得到训练好的模型；
4. 预测新数据：运行“5.模型预测模块/lstm_predict.py”，输入最新数据获取结果。

### 5.3 结果解读示例
=== 信用债走势预测结果 ===
1. 预测基准日期：2024-06-10
2. 预测周期：未来5天
3. 预测结论：看多
4. 各类别概率：看空15%、看多68%、震荡17%
5. 投资建议：建议增持信用债（预计收益率将下跌，价格可能上涨）

- 若“看多概率”显著高于其他类别（如>60%），信号更可靠；
- 若三类概率接近（如30%-40%-30%），建议观望（模型判断模糊）。


## 六、注意事项与优化建议
### 6.1 关键注意事项
1. 数据质量优先：
   - 原始数据需包含完整的日期列和指标值，缺失值过多会影响模型效果；
   - 新增指标时，确保与现有数据频率一致（如统一为日频）。

2. 避免数据泄露：
   - 严格按时间拆分训练/验证/测试集（禁止随机打乱）；
   - 标准化仅用训练集的均值/标准差，禁止用全量数据拟合。

3. 模型局限性：
   - 对“黑天鹅事件”（如突发政策调整）预测能力有限，需结合人工判断；
   - 建议每1-3个月用最新数据重新训练模型，适应市场变化。

### 6.2 优化方向
1. 模型升级：
   - 增加LSTM层数（如2层LSTM）捕捉更复杂的模式；
   - 引入注意力机制（Attention），让模型自动关注关键指标（如央行政策相关数据）。

2. 功能扩展：
   - 对接实时行情接口，实现分钟级更新预测；
   - 增加多场景预测（如不同信用评级、不同期限的信用债）。

3. 可解释性提升：
   - 用SHAP值分析模型关注的核心指标（如“美元指数对信用债走势的影响权重”）；
   - 生成“指标-走势”关联规则（如“GDP增速>6.5%时，70%概率看多”）。
